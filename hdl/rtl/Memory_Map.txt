================================================================================
RISC-V SYSTEM MEMORY MAP
================================================================================

This document describes the memory address map for the AI RISC-V KC32 System.

================================================================================
MEMORY REGIONS (CORE & BUS)
================================================================================

1. DEBUG MODULE
   Address Range:  0x0000_0000 - 0x0000_0FFF
   Size:           4 KB
   Description:    RISC-V Debug Module address space (Registers, Program Buffer, ROM)
   Access:         Read/Write

2. BOOT ROM (MROM)
   Address Range:  0x0000_1000 - 0x0000_7FFF
   Size:           28 KB Window (256 bytes actual ROM data at offset 0)
   Description:    Primary bootloader code. Reset PC points to 0x0000_1000.
   Access:         Read-Only

3. CLINT (Core Local Interruptor)
   Address Range:  0x0200_0000 - 0x0200_FFFF
   Size:           64 KB
   Description:    Standard RISC-V Timer (MTIME) and SW Interrupts (MSIP).
   Access:         Read/Write

4. PLIC (Platform-Level Interrupt Controller)
   Address Range:  0x0C00_0000 - 0x0C03_FFFF
   Size:           256 KB
   Description:    External interrupt management logic.
   Access:         Read/Write

5. INSTRUCTION RAM (IRAM)
   Address Range:  0x8000_0000 - 0x8FFF_FFFF
   Size:           256 MB (Window), 16 KB (Actual)
   Description:    Core-internal instruction memory. 
                   Accessed directly by IFU (hardcoded check in riscv_axi_ifetch.v).
   Access:         Read-Only (Fetch) / Read-Write (via Data Bus if enabled)

6. DATA RAM (DRAM)
   Address Range:  0x9000_0000 - 0x9FFF_FFFF
   Size:           256 MB (Window), 16 KB (Actual)
   Description:    Core-internal data memory.
                   Accessed directly by Data Unit (hardcoded check in riscv_axi_data.v).
   Access:         Read/Write

================================================================================
PERIPHERAL SPACE (APB DECODER)
================================================================================
Base Address: 0x1000_0000
Window Size:  0x0001_0000 (64 KB)

Peripheral Slot Map (4KB per slot):

Slot 0: UART 16550A
   Address Range:  0x1000_0000 - 0x1000_0FFF
   Description:    Serial communication interface.

Slot 1: RESERVED
   Address Range:  0x1000_1000 - 0x1000_1FFF
   Description:    Reserved space (Previously ADC/DAC). Returns 0x00000000.

Slot 2: GPIO (General Purpose I/O)
   Address Range:  0x1000_2000 - 0x1000_2FFF
   Description:    32-bit configurable I/O pins, drive strength, and interrupts.

Slots 3-15: RESERVED
   Address Range:  0x1000_3000 - 0x1001_0000 (End of Peripheral AXI)
   Description:    Unmapped address space. Access returns PSLVERR.

================================================================================
ADDRESS ROUTING LOGIC (riscv32KC2_bus.v)
================================================================================

Instruction Fetch (AXI Port S0):
  - 0x8000_0000 - 0x8FFF_FFFF -> Internal to Core (IRAM)
  - 0x0000_1000 - 0x0000_7FFF -> Routes to MROM (Master 2)
  - 0x0000_0000 - 0x0000_0FFF -> Routes to Debug (Master 3)
  - Addresses outside these ranges will result in AXI Decode Error (DECERR).

Data Access (AXI Port S1):
  - 0x8000_0000 - 0x8FFF_FFFF -> Internal to Core (IRAM)
  - 0x9000_0000 - 0x9FFF_FFFF -> Internal to Core (DRAM)
  - 0x0000_0000 - 0x0000_0FFF -> Routes to Debug (Master 3)
  - 0x1000_0000 - 0x1000_FFFF -> Routes to Peripherals (Master 4)
  - 0x0200_0000 - 0x0201_0000 -> Routes to CLINT (Master 5)
  - 0x0C00_0000 - 0x0C03_FFFF -> Routes to PLIC (Master 6)
  - All other address space returns AXI Decode Error.

================================================================================
IMPLEMENTATION NOTES
================================================================================

1. Reset PC: 0x0000_1000 (Boot ROM).
2. mtvec: Generally programmed to an address in IRAM (0x8000_....).
3. Misaligned access: Hardware detects misaligned Load/Store and generates exceptions.
4. Instruction Fetch Fallback: The core can only fetch instructions from addresses 
   explicitly mapped in the core's internal mux or the S0 (IF) bus decoder.
